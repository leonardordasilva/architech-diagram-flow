Marca d'Água, Exportação Condicional e Modo Read-Only
Objetivo: Implementar os 3 mecanismos de diferenciação visual que não existem na aplicação — marca d'água nas exportações, bloqueio condicional de formatos de exportação, e modo de visualização read-only para links compartilhados — de forma que o plano Free tenha restrições visíveis e o upgrade para Pro seja tangível.

Contexto: O useExportHandlers.ts exporta PNG e SVG via html-to-image (funções toPng e toSvg) sem nenhuma pós-processamento. O Toolbar.tsx renderiza todos os botões de exportação igualmente para todos os usuários. O SharedDiagram.tsx carrega o DiagramCanvas completo com edição total para qualquer visitante, autenticado ou não.

Dependência: Este PRD depende do PRD 1 (hook usePlanLimits e planStore disponíveis).

1. Marca d'Água na Exportação PNG (Plano Free)
1.1 Comportamento Esperado
Quando um usuário do plano Free exporta PNG, a imagem final deve conter o texto "Made with MicroFlow Architect" posicionado no canto inferior direito, com fonte semi-transparente (opacity 0.5), tamanho 14px, cor branca no dark mode e cinza escuro no light mode. O texto deve ser visível mas não obstruir o conteúdo principal do diagrama. Usuários Pro e Team não devem ter marca d'água.

1.2 Implementação Técnica
Alterar handleExportPNG em useExportHandlers.ts. Após a chamada a toPng() que retorna o data URL, verificar planStore.watermarkEnabled. Se true: criar elemento <canvas> em memória, criar Image() com src = dataUrl, no onload desenhar a imagem no canvas com drawImage(), posicionar o texto via ctx.font = '14px DM Sans, sans-serif', ctx.globalAlpha = 0.5, ctx.fillStyle condicional ao darkMode, ctx.textAlign = 'right', ctx.fillText('Made with MicroFlow Architect', canvas.width - 16, canvas.height - 16), e gerar novo data URL via canvas.toDataURL('image/png'). Se false: comportamento atual sem alteração.

1.3 Marca d'Água na Exportação SVG (Plano Free não exporta SVG)
Não aplicável — o plano Free não tem acesso a SVG. A marca d'água é apenas no PNG.

2. Bloqueio Condicional de Formatos de Exportação
2.1 Comportamento Esperado
No plano Free, os botões de exportação SVG e Mermaid devem estar visíveis na toolbar mas desabilitados, com ícone de cadeado sobreposto e tooltip "Disponível no plano Pro". O clique nesses botões deve abrir o UpgradeModal (definido no PRD 1) em vez de executar a exportação. Os botões de PNG e JSON permanecem funcionais. Nos planos Pro e Team, todos os botões funcionam normalmente.

2.2 Implementação Técnica
Alterar o dropdown de exportação no Toolbar.tsx. Cada DropdownMenuItem de exportação deve receber a prop disabled condicional: consultar planStore.allowedExportFormats e verificar se o formato está na lista. Itens bloqueados: renderizar com className="opacity-50", ícone Lock (já disponível no import do Lucide) ao lado do texto, e onClick que chama onUpgrade('export_svg') ou onUpgrade('export_mermaid') em vez da função de exportação. Nova prop onUpgrade no Toolbar que o DiagramCanvas conecta ao UpgradeModal.

2.3 Exportação JSON
Permanece disponível em todos os planos. Sem alteração.

3. Modo Read-Only para Links Compartilhados
3.1 Comportamento Esperado
Quando um visitante acessa /diagram/:shareToken, o diagrama deve abrir em modo visualização: pode ver o diagrama, fazer zoom, pan e navegar, mas não pode editar nós, criar conexões, deletar elementos, exportar, ou salvar. A toolbar completa deve ser substituída por uma toolbar simplificada com apenas o nome do diagrama (não editável), botão de dark/light mode, e um CTA "Crie sua conta para editar". Se o visitante estiver autenticado e for owner ou colaborador registrado em diagram_shares, o modo completo de edição é mantido (comportamento atual).

3.2 Implementação Técnica — SharedDiagram.tsx
Após carregar o diagrama via loadDiagramByToken, determinar o modo: se o usuário está autenticado, verificar se user.id === diagram.owner_id OU se existe registro em diagram_shares para este user.id + diagram.id (nova RPC is_diagram_collaborator). Se sim: modo edição (comportamento atual). Se não (ou se não autenticado): modo read-only.

3.3 Implementação Técnica — DiagramCanvas.tsx
Nova prop readOnly?: boolean (default false). Quando readOnly === true: o componente <Toolbar> é substituído pelo novo componente <ReadOnlyToolbar> (nome do diagrama + dark mode + CTA). O <ReactFlow> recebe nodesDraggable={false}, nodesConnectable={false}, elementsSelectable={false}. Os handlers onNodesChange, onEdgesChange, onConnect são substituídos por no-ops. O handler onNodeContextMenu é no-op. A StatusBar exibe "Modo visualização" em vez de contadores de save. O DiagramHeader oculta botões de save, refresh e logout, e exibe badge "Visualização" junto com CTA "Criar conta".

3.4 Componente ReadOnlyToolbar
Renderiza: nome do diagrama (texto estático, não Input), separador, botão dark/light mode, separador, botão "Criar conta para editar" que navega para /app (onde o AuthPage será exibido se não autenticado). Estilização idêntica à toolbar atual para consistência visual.

3.5 RPC is_diagram_collaborator
Nova função PostgreSQL: recebe p_user_id e p_diagram_id, retorna boolean. Query: SELECT EXISTS(SELECT 1 FROM diagrams WHERE id = p_diagram_id AND owner_id = p_user_id AND deleted_at IS NULL) OR EXISTS(SELECT 1 FROM diagram_shares WHERE diagram_id = p_diagram_id AND shared_with_id = p_user_id).

4. Critérios de Aceite
A exportação PNG no plano Free deve conter marca d'água legível no canto inferior direito. A exportação PNG nos planos Pro e Team deve ser limpa, sem marca d'água. Os botões SVG e Mermaid no plano Free devem estar visíveis, desabilitados, com cadeado. O clique nos botões bloqueados deve abrir o modal de upgrade. Um visitante não autenticado acessando /diagram/:shareToken deve ver o diagrama em modo read-only sem toolbar de edição. Um visitante autenticado que é owner ou colaborador acessando /diagram/:shareToken deve ter edição completa. O CTA "Criar conta para editar" deve navegar para a página de autenticação.
