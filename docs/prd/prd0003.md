Este PRD cobre exclusivamente os itens identificados na reanálise pós-v2.0. São 7 épicos organizados em três categorias: correções de bugs críticos (mutação direta de data em ServiceNode, protocolo não atribuído ao criar aresta), melhorias de funcionalidades existentes (Meus Diagramas com React Query, protocolo inferido pela IA, CORS com fallback de dev), e refinamentos de qualidade (cor do Kafka, proteção de rota, feedback de fork em diagrama compartilhado).
Regras que se aplicam a toda implementação:
•	Preservar 100% das funcionalidades existentes sem nenhuma regressão.
•	Todo texto de UI em português brasileiro.
•	Todo componente novo com suporte a dark mode via classes Tailwind dark:.
•	TypeScript estrito — sem novos any sem justificativa comentada.
•	Estrutura de arquivos dentro de /src, seguindo o padrão estabelecido.

ÉPICO 1 — Bug Crítico: Corrigir Mutação Direta de data em ServiceNode
Prioridade: 🔴 CRÍTICA — Implementar primeiro.
Contexto do problema: O ServiceNode.tsx contém dois antipadrões de mutação direta do objeto data do nó React Flow:
Copy// PROBLEMA 1 — em handleBlur:
nodeData.label = label; // mutação direta — não dispara re-render nem captura no zundo

// PROBLEMA 2 — em handleDbRename:
nodeData.internalDatabases[index] = newName; // idem
// e em handleSvcRename:
nodeData.internalServices[index] = newName; // idem
Isso significa que renomear um nó por duplo clique no canvas não é capturado pelo sistema de undo/redo do zundo, porque o store nunca recebe uma nova referência de objeto. Também pode causar inconsistências entre o estado visual e o estado real do store.
O que deve ser implementado:
Os nós do React Flow não devem modificar seu próprio data diretamente. Toda mutação deve passar pelo useDiagramStore.getState().setNodes(...). Como os componentes de nó no React Flow não recebem dispatch diretamente, a solução correta é usar o hook useReactFlow para obter setNodes, ou chamar diretamente useDiagramStore.getState() de dentro do nó.
Refatorar ServiceNode.tsx substituindo os três pontos de mutação direta pela atualização imutável via store:
Copy// src/components/nodes/ServiceNode.tsx
// Substituir as três funções de mutação:

const handleBlur = () => {
  setEditing(false);
  // Atualiza imutavelmente via store
  useDiagramStore.getState().setNodes(
    useDiagramStore.getState().nodes.map((n) =>
      n.id === id ? { ...n, data: { ...n.data, label } } : n
    )
  );
};

const handleDbRename = (index: number, newName: string) => {
  const currentDbs = nodeData.internalDatabases ?? [];
  const updated = currentDbs.map((db, i) => (i === index ? newName : db));
  useDiagramStore.getState().setNodes(
    useDiagramStore.getState().nodes.map((n) =>
      n.id === id ? { ...n, data: { ...n.data, internalDatabases: updated } } : n
    )
  );
};

const handleSvcRename = (index: number, newName: string) => {
  const currentSvcs = nodeData.internalServices ?? [];
  const updated = currentSvcs.map((svc, i) => (i === index ? newName : svc));
  useDiagramStore.getState().setNodes(
    useDiagramStore.getState().nodes.map((n) =>
      n.id === id ? { ...n, data: { ...n.data, internalServices: updated } } : n
    )
  );
};

export function inferProtocol(sourceType: NodeType, targetType: NodeType): EdgeProtocol {
  if (sourceType === 'service' && targetType === 'service')  return 'REST';
  if (sourceType === 'service' && targetType === 'database') return 'TCP';
  if (sourceType === 'service' && targetType === 'queue')    return 'AMQP';
  if (sourceType === 'queue'   && targetType === 'service')  return 'Kafka';
  if (sourceType === 'service' && targetType === 'external') return 'HTTPS';
  if (sourceType === 'external' && targetType === 'service') return 'REST';
  if (sourceType === 'queue'   && targetType === 'queue')    return 'AMQP';
  return 'REST'; // fallback
}

ÉPICO 6 — Correção: Proteção de Rota /my-diagrams com ProtectedRoute

import { Navigate } from 'react-router-dom';
import { useAuth } from '@/hooks/useAuth';

interface ProtectedRouteProps {
  children: React.ReactNode;
  redirectTo?: string;
}

export default function ProtectedRoute({ children, redirectTo = '/' }: ProtectedRouteProps) {
  const { user, loading } = useAuth();

  // Aguarda a sessão ser carregada antes de redirecionar
  if (loading) {
    return (
      <div className="flex min-h-screen items-center justify-center bg-background">
        <p className="text-muted-foreground text-sm">Carregando...</p>
      </div>
    );
  }

  if (!user) {
    return <Navigate to={redirectTo} replace />;
  }

  return <>{children}</>;
}
