## TAREFA T1 — Adicionar try/catch em `ImportJSONModal`

**Arquivo:** `src/components/ImportJSONModal.tsx`
**Pilar:** Correctness / UX
**Risco que resolve:** Exceção não capturada ao importar JSON malformado

### Problema

```typescript
// ImportJSONModal.tsx — situação atual (linha ~38)
const parsed = JSON.parse(jsonText); // ← sem try/catch
```

Se o usuário colar um JSON inválido (texto corrompido, JSON5, YAML, etc.), `JSON.parse` lança `SyntaxError`. Sem tratamento, a exceção se propaga, possivelmente causando freeze ou erro silencioso na UI.

### O que fazer

**Passo 1 — Envolver `JSON.parse` em try/catch e validar com schema Zod:**

```typescript
// DEPOIS — ImportJSONModal.tsx
import { ImportDiagramSchema } from '@/schemas/diagramSchema'; // ou schema existente

function handleImport() {
  // 1. Parse JSON com tratamento de erro
  let parsed: unknown;
  try {
    parsed = JSON.parse(jsonText.trim());
  } catch {
    toast({
      title: t('importModal.invalidJson'),
      description: t('importModal.invalidJsonDescription'),
      variant: 'destructive',
    });
    return;
  }

  // 2. Validar estrutura com Zod (já deve existir um schema)
  const result = ImportDiagramSchema.safeParse(parsed);
  if (!result.success) {
    toast({
      title: t('importModal.invalidStructure'),
      description: t('importModal.invalidStructureDescription'),
      variant: 'destructive',
    });
    return;
  }

  // 3. Aplicar ao store
  const { nodes, edges } = result.data;
  setNodes(nodes);
  setEdges(edges);
  onClose();
}
```

**Passo 2 — Adicionar chaves de tradução (`src/i18n/`):**

```json
  "importModal": {

    "invalidJsonDescription": "O texto colado não é um JSON válido. Verifique a formatação e tente novamente.",

    "invalidStructureDescription": "O JSON é válido, mas não corresponde ao formato de diagrama esperado."
  }

```

**Passo 3 — Verificar se `ImportDiagramSchema` já existe:**

- Se existir em `src/schemas/diagramSchema.ts`, importar e usar.
- Se não existir, criar schema mínimo:

```typescript
// src/schemas/diagramSchema.ts — adicionar se não existir
export const ImportDiagramSchema = z.object({
  nodes: z.array(DiagramNodeSchema),
  edges: z.array(DiagramEdgeSchema),
});
```

### Critérios de Aceite

- [ ] `JSON.parse` está dentro de try/catch em `ImportJSONModal`
- [ ] JSON inválido exibe toast de erro sem crashar a aplicação
- [ ] Estrutura de diagrama inválida (JSON válido mas sem nodes/edges) também exibe toast
- [ ] Todos os locales têm as chaves de tradução adicionadas
- [ ] Teste em `ImportJSONModal.test.tsx` cobre: JSON inválido, estrutura inválida, importação bem-sucedida

## TAREFA T2 — Teste de integração broadcast → DB save → Realtime

**Arquivo:** `src/hooks/useRealtimeCollab.test.ts`
**Pilar:** Qualidade / Testes
**Risco que resolve:** Race condition entre broadcast e Postgres Realtime não detectada automaticamente

### Problema

O hook `useRealtimeCollab` mantém dois canais simultâneos:

1. **Broadcast** (linhas ~80-156): dados enviados diretamente para colaboradores online via `channel.send()`
2. **Postgres Realtime** (linhas ~163-243): evento `UPDATE` do banco chegando após save persistido

Fluxo problemático não testado:
```
T1: Cliente A faz save → envia broadcast com nodes/edges
T2: Cliente B recebe broadcast → aplica ao store (revision local = N)
T3: Banco persiste → Postgres Realtime entrega evento UPDATE
T4: Cliente B recebe Realtime com revision_number > lastRevisionRef
T5: Cliente B aplica novamente → nodes aparecem duas vezes? ou descarta?
```

Sem teste cobrindo este fluxo, regressões nesta área passam despercebidas.

### O que fazer

**Adicionar describe de integração em `src/hooks/useRealtimeCollab.test.ts`:**

```typescript
describe('integração: broadcast seguido de Realtime DB update', () => {

  it('aplica broadcast imediatamente e ignora Realtime subsequente com mesmo conteúdo', async () => {
    // Arrange
    const { result } = renderHook(() => useRealtimeCollab({ diagramId: 'diag-1' }));
    const nodes = [mockNode('node-1')];
    const edges = [mockEdge('edge-1')];

    // Act 1: broadcast chega
    await act(() => simulateBroadcast(result, { nodes, edges, updatedAt: '2026-04-09T10:00:00.000Z' }));
    const stateAfterBroadcast = getCurrentNodes(result);

    // Act 2: Realtime chega com mesmo conteúdo, revision_number = 1
    await act(() => simulateRealtimeUpdate(result, {
      nodes,
      edges,
      revision_number: 1,
      updated_at: '2026-04-09T10:00:00.000Z',
    }));
    const stateAfterRealtime = getCurrentNodes(result);

    // Assert: estado não duplicou
    expect(stateAfterRealtime).toHaveLength(stateAfterBroadcast.length);
  });

  it('aplica Realtime quando traz conteúdo mais novo que o broadcast', async () => {
    const { result } = renderHook(() => useRealtimeCollab({ diagramId: 'diag-1' }));
    const nodesV1 = [mockNode('node-1')];
    const nodesV2 = [mockNode('node-1'), mockNode('node-2')]; // versão mais nova

    // Act 1: broadcast com V1
    await act(() => simulateBroadcast(result, { nodes: nodesV1, edges: [] }));

    // Act 2: Realtime com V2, revision_number maior
    await act(() => simulateRealtimeUpdate(result, {
      nodes: nodesV2,
      edges: [],
      revision_number: 2,
    }));

    // Assert: estado foi atualizado para V2
    expect(getCurrentNodes(result)).toHaveLength(2);
  });

  it('ignora Realtime com revision_number já visto', async () => {
    const { result } = renderHook(() => useRealtimeCollab({ diagramId: 'diag-1' }));

    // Act: Realtime com revision 1 (aplicado)
    await act(() => simulateRealtimeUpdate(result, {
      nodes: [mockNode('node-1')],
      edges: [],
      revision_number: 1,
    }));

    // Act: mesmo revision_number chega novamente (rede duplicada)
    await act(() => simulateRealtimeUpdate(result, {
      nodes: [mockNode('node-1'), mockNode('node-extra')], // conteúdo diferente, mesmo revision
      edges: [],
      revision_number: 1,
    }));

    // Assert: segundo update foi ignorado (não aplicado)
    expect(getCurrentNodes(result)).toHaveLength(1);
  });

  it('pausa undo/redo durante aplicação de update remoto', async () => {
    const { result } = renderHook(() => useRealtimeCollab({ diagramId: 'diag-1' }));
    const pauseSpy = vi.spyOn(useDiagramStore.getState(), 'pauseHistory');
    const resumeSpy = vi.spyOn(useDiagramStore.getState(), 'resumeHistory');

    await act(() => simulateRealtimeUpdate(result, {
      nodes: [mockNode('node-1')],
      edges: [],
      revision_number: 1,
    }));

    expect(pauseSpy).toHaveBeenCalledBefore(resumeSpy);
    expect(resumeSpy).toHaveBeenCalledOnce();
  });

  it('trata Realtime com dados criptografados e descriptografa antes de aplicar', async () => {
    const { result } = renderHook(() => useRealtimeCollab({ diagramId: 'diag-1' }));

    // Simular payload criptografado vindo do banco
    const encryptedPayload = { iv: 'mock-iv', ciphertext: 'mock-cipher' };
    const decryptedNodes = [mockNode('node-decrypted')];
    mockDecryptDiagramData.mockResolvedValueOnce({
      nodes: decryptedNodes,
      edges: [],
    });

    await act(() => simulateRealtimeUpdate(result, {
      nodes: encryptedPayload, // não é array — está criptografado
      edges: encryptedPayload,
      revision_number: 1,
    }));

    expect(mockDecryptDiagramData).toHaveBeenCalledOnce();
    expect(getCurrentNodes(result)).toEqual(decryptedNodes);
  });

  it('descarta update criptografado quando descriptografia falha', async () => {
    const { result } = renderHook(() => useRealtimeCollab({ diagramId: 'diag-1' }));
    mockDecryptDiagramData.mockRejectedValueOnce(new Error('Decryption failed'));

    const initialNodes = getCurrentNodes(result);
    await act(() => simulateRealtimeUpdate(result, {
      nodes: { iv: 'bad', ciphertext: 'bad' },
      edges: { iv: 'bad', ciphertext: 'bad' },
      revision_number: 2,
    }));

    // Assert: store não foi modificado
    expect(getCurrentNodes(result)).toEqual(initialNodes);
  });

});
```

**Helpers de teste (adicionar em setup ou no próprio arquivo):**

```typescript
// Helpers para simular eventos do canal Supabase nos testes
function simulateBroadcast(result, payload) {
  const channel = getLastCreatedChannel();
  channel.simulateEvent('broadcast', { event: 'diagram_update', payload });
}

function simulateRealtimeUpdate(result, newRecord) {
  const channel = getLastCreatedChannel();
  channel.simulateEvent('postgres_changes', {
    eventType: 'UPDATE',
    new: newRecord,
  });
}

function getCurrentNodes(result) {
  return useDiagramStore.getState().nodes;
}
```
