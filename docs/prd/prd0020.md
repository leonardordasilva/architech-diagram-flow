# MicroFlow Architect

## Visão geral do produto

MicroFlow Architect é um editor visual de diagramas de arquitetura de microsserviços
que roda inteiramente no navegador. Ele permite a engenheiros de software, arquitetos
de solução e tech leads modelar, analisar e compartilhar diagramas de sistemas
distribuídos com colaboração em tempo real, persistência criptografada na nuvem e
exportação profissional.

O produto está em produção, hospedado em Vercel + Supabase, com interface disponível
em português (pt-BR) e inglês (en), e score de auditoria técnica de 99/100.

- Arquitetos de software e solução em empresas de médio e grande porte.

- Times de engenharia que usam Kafka, RabbitMQ, IBM MQ, gRPC e REST no dia-a-dia.

## Funcionalidades (organizadas por seção da landing page)

### 1. Editor Visual Drag-and-Drop
O canvas interativo permite arrastar e posicionar nós livremente. Cada nó representa
um componente real de arquitetura. As conexões entre nós seguem regras de validação
(ex: um banco de dados não pode se conectar diretamente a uma fila). O canvas suporta
zoom, pan, minimap, snap-guides de alinhamento e grid de encaixe.

Elementos visuais do editor:
- Toolbar superior com nome do diagrama, botões de adicionar nós, undo/redo,
  layout automático, exportação e modo claro/escuro.
- Status bar inferior com contagem de nós, conexões e nível de zoom.
- Painel lateral de propriedades ao clicar em um nó.
- Menu de contexto (clique direito) para criar nós a partir de outro.
- Legenda de protocolos com cores distintas (síncrono vs assíncrono).

### 2. Quatro tipos de nós especializados

**Microserviço (azul, ícone Box)**
Representa um serviço independente. Suporta bancos de dados internos embarcados
(Oracle, Redis, PostgreSQL, MongoDB, MySQL, SQL Server, DynamoDB, Cassandra) e
bibliotecas internas. Label editável com duplo-clique.

**Banco de Dados (cor varia por tipo, ícone Database)**
Representa um banco externo ao serviço. Oito variantes com cores únicas:
Oracle (vermelho), PostgreSQL (azul escuro), MySQL (laranja), SQL Server (cinza),
MongoDB (verde), Redis (vermelho vibrante), DynamoDB (azul Amazon), Cassandra (teal).

**Fila/Mensageria (verde, ícone varia)**
Representa um sistema de mensageria. Três variantes: IBM MQ (ícone envelope),
Kafka (ícone antena), RabbitMQ (ícone balão). Conexões de/para filas são exibidas
com linha tracejada para diferenciar de chamadas síncronas.

**Sistema Externo (cinza, borda tracejada, ícone varia)**
Representa APIs e serviços fora do domínio. Sete categorias com ícones distintos:
API (globo), CDN (wifi), Auth (escudo), Payment (cartão), Storage (database),
Analytics (gráfico), Other (cubo). Suporta protocolos: REST, gRPC, GraphQL,
WebSocket, HTTPS.

### 3. Conexões inteligentes com protocolos

Cada conexão entre nós pode receber um protocolo visual (modal de seleção ao
conectar). Os protocolos disponíveis são:

| Protocolo | Tipo       | Cor visual     |
|-----------|------------|----------------|
| REST      | Síncrono   | Azul           |
| gRPC      | Síncrono   | Roxo           |
| GraphQL   | Síncrono   | Rosa           |
| WebSocket | Assíncrono | Laranja        |
| Kafka     | Assíncrono | Verde escuro   |
| AMQP      | Assíncrono | Teal           |
| MQTT      | Assíncrono | Amarelo        |
| HTTPS     | Síncrono   | Verde claro    |
| TCP       | Síncrono   | Cinza escuro   |
| UDP       | Assíncrono | Cinza claro    |
| SQL       | Síncrono   | Dourado        |

As conexões são edges ortogonais (ângulos de 90°) editáveis — o usuário pode
arrastar segmentos horizontais e verticais para ajustar o caminho. A legenda
aparece automaticamente no canvas quando há protocolos atribuídos.

### 4. Undo/Redo completo
Histórico de até 50 estados com Ctrl+Z / Ctrl+Y. O undo captura: adição/remoção
de nós, movimentação, renomeação, mudanças de conexão e renomeação do diagrama.
Atualizações remotas de colaboradores NÃO entram no histórico local — garantindo
que o undo nunca desfaz o trabalho de outra pessoa.

### 5. Exportação profissional
- **PNG** — imagem rasterizada com fundo (claro ou escuro) e padding automático.
- **SVG** — imagem vetorial para uso em documentação.
- **Mermaid** — código Mermaid.js colável em qualquer markdown (GitHub, Notion,
  Confluence, etc.).
- **JSON** — formato estruturado importável de volta na ferramenta.

### 6. Importação de JSON
Arraste um arquivo .json ou cole o conteúdo. A validação com Zod garante que
somente diagramas no formato correto sejam importados.

### 7. Autenticação e persistência na nuvem
Conta pessoal via e-mail/senha com confirmação por e-mail e recuperação de senha.
Diagramas salvos na nuvem com criptografia AES-256-GCM (os dados são
criptografados antes de chegar ao banco). Listagem paginada em "Meus Diagramas"
com ações de renomear, duplicar, compartilhar e excluir.

### 8. Compartilhamento e colaboração em tempo real
Compartilhamento por busca de e-mail (typeahead com debounce). O destinatário vê
o diagrama em "Compartilhados comigo" e pode editá-lo. As alterações são
sincronizadas em tempo real via WebSocket — avatares dos colaboradores online
aparecem no header com cores distintas.

### 9. Auto-save com recuperação
O diagrama é salvo automaticamente a cada 1.5 segundos de inatividade usando
compressão gzip em IndexedDB. Se o navegador fechar inesperadamente, ao reabrir
o app exibe um banner "Diagrama recuperado — salvo em [data/hora]" com opções de
restaurar ou descartar.

### 10. Dark Mode / Light Mode
Alternância com um clique na toolbar. A preferência é persistida em localStorage
e respeita o preço de sistema do SO na primeira visita.

### 11. Atalhos de teclado
| Atalho         | Ação                     |
|----------------|--------------------------|
| Ctrl + Z       | Desfazer                 |
| Ctrl + Y       | Refazer                  |
| Ctrl + Shift+Z | Refazer (alternativo)    |
| Delete / Backsp| Deletar selecionado      |
| Ctrl + S       | Salvar na nuvem          |
| Ctrl + A       | Selecionar todos os nós  |
| Escape         | Fechar modais            |
| ?              | Abrir atalhos de teclado |

### 12. Internacionalização
Interface disponível em português (Brasil) e inglês. O idioma é detectado
automaticamente pelo navegador. São 234 strings traduzidas cobrindo toda a
interface, incluindo mensagens de erro, toasts e labels de nós.

## Dados técnicos relevantes para a landing page

### Onde roda
100% no navegador. Sem instalação. Funciona em Chrome, Edge, Firefox e Safari
(desktop). Otimizado para telas a partir de 1024px.

### Segurança (para seção de confiança/trust)
- Criptografia AES-256-GCM em todos os diagramas armazenados.
- Autenticação via Supabase Auth com verificação de e-mail.
- Políticas de acesso granulares (Row Level Security) no banco de dados.
- Validação de todos os inputs com Zod.
- Content Security Policy restritiva no HTML.

## Estrutura de telas da aplicação (referência visual para o front-end)

### Tela 1 — Login / Cadastro / Recuperação de senha
Formulário centralizado com card em fundo clean. Logo "MicroFlow Architect" no
topo. Três estados: login (e-mail + senha + link "esqueceu?"), cadastro (e-mail
+ senha + link "já tem conta?"), recuperação (e-mail + botão enviar). Transição
entre estados sem navegação de página.

### Tela 2 — Editor (tela principal)
Canvas de diagramação em tela cheia. Barra de ferramentas no topo com: input de
nome do diagrama, dropdowns para adicionar nós (Microserviço, Banco de Dados,
Fila, API), botões de ação (deletar, limpar, undo, redo), dropdown de layout
automático, dropdown de exportação, botão de importação JSON, toggle dark/light.
Header com badges de status ("Editando diagrama compartilhado", "Cópia salva"),
avatares de colaboradores, botões de salvar/atualizar/sair. Status bar no rodapé
com contagem de nós, conexões e zoom. Painel lateral de propriedades (abre ao
clicar em um nó). Minimap no canto inferior direito.

### Tela 3 — Meus Diagramas
Grid de cards com: título, contagem "X nós · Y conexões", data de atualização.
Ações por card: compartilhar, duplicar, renomear, excluir. Seção "Compartilhados
comigo" abaixo com badges indicando o e-mail do proprietário. Botão "Novo Diagrama"
no topo. Paginação com "Carregar mais".

### Tela 4 — Diagrama compartilhado (via link)
Mesma interface do editor, mas com badge "Visualizando diagrama compartilhado" e
sem permissões de dono. Após salvar, o diagrama aparece como cópia em "Meus
Diagramas" do colaborador.

### Tela 5 — Redefinição de senha
Formulário com dois campos (nova senha + confirmação) e validação inline (mínimo
6 caracteres, senhas devem coincidir).

### Tela 6 — 404
Mensagem "Página não encontrada" com link para voltar ao início.

## Observações para o front-end

- Utilizar a seguinte stack: 
| Camada            | Recomendação                     |
|-------------------|----------------------------------|
| Meta-framework    | React/Next.js                    |
| Estilização       | Tailwind CSS                     |
| Componentes base  | shadcn/ui                        |
| Estética avançada | Aceternity UI, Magic UI          |
| Animações         | Framer Motion / Motion           |

- O CTA deve apontar para a URL do app em produção (a ser definida pelo PO).
- A landing page precisa funcionar bem em mobile (o editor em si é desktop-first,
  mas a landing deve ser responsiva).
- Considerar animações sutis no hero (ex: canvas do editor animando com nós se
  conectando automaticamente) para transmitir a experiência do produto.
- A landing page deve ser bilíngue (pt-BR e en) ou pelo menos ter a versão em
  inglês, dado que o produto já suporta ambos os idiomas.
- Os screenshots/demos do editor devem ser capturados em ambos os temas
  (claro e escuro) para uso nas seções da landing.
- Utilize as imagens disponíveis em /src/img onde for necessário para a criação da landing page.
- O favicon deve ser utilizado no favicon da landing page.
