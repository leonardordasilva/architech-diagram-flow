ITEM 1 — Adicionar validação fail-fast no supabase client
Arquivo: src/integrations/supabase/client.ts

O que fazer: Adicionar validação das variáveis de ambiente VITE_SUPABASE_URL e VITE_SUPABASE_PUBLISHABLE_KEY com throw de erro descritivo antes da criação do client. Isso garante fail-fast no bootstrap da aplicação se as variáveis não estiverem configuradas, em vez de erros crípticos posteriores.

Substituir o conteúdo atual:

Copyconst SUPABASE_URL = import.meta.env.VITE_SUPABASE_URL;
const SUPABASE_PUBLISHABLE_KEY = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY;
Por:

Copyconst SUPABASE_URL = import.meta.env.VITE_SUPABASE_URL;
const SUPABASE_PUBLISHABLE_KEY = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY;

if (!SUPABASE_URL) {
  throw new Error(
    'Missing VITE_SUPABASE_URL environment variable. ' +
    'Please set it in your .env file or deployment configuration.'
  );
}

if (!SUPABASE_PUBLISHABLE_KEY) {
  throw new Error(
    'Missing VITE_SUPABASE_PUBLISHABLE_KEY environment variable. ' +
    'Please set it in your .env file or deployment configuration.'
  );
}
Manter o restante do arquivo exatamente como está. O createClient<Database>(SUPABASE_URL, SUPABASE_PUBLISHABLE_KEY, ...) continua inalterado — após a validação acima, o TypeScript infere que ambos os valores não são undefined.

Critério de aceite: Se VITE_SUPABASE_URL ou VITE_SUPABASE_PUBLISHABLE_KEY não estiverem definidas, a aplicação falha imediatamente no bootstrap com mensagem de erro clara indicando qual variável está faltando. Build compila sem erros.
